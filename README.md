## 景文件夹命名规则

景文件夹名 = 完整 Scene ID，由下划线分隔的多个字段组成。例如：

```
LC08_L2SP_119038_20250415_20250424_02_T1_SR
│     │     │       │         │       │  │  │
│     │     │       │         │       │  │  └── 第8段：产品后缀
│     │     │       │         │       │  └───── 第7段：质量等级
│     │     │       │         │       └──────── 第6段：集合版本号
│     │     │       │         └──────────────── 第5段：处理日期
│     │     │       └────────────────────────── 第4段：采集日期
│     │     └────────────────────────────────── 第3段：WRS-2 Path/Row
│     └──────────────────────────────────────── 第2段：处理级别
└────────────────────────────────────────────── 第1段：卫星编号
```

各字段详细含义：

| 段序 | 字段 | 示例值 | 含义 |
|------|------|--------|------|
| 第1段 | 卫星编号 | `LC08` | Landsat 8；`LC09` = Landsat 9 |
| 第2段 | 处理级别 | `L1TP` / `L1GT` / `L2SP` | 见下方"处理级别说明" |
| 第3段 | Path/Row | `119038` | WRS-2 轨道编号，前3位 = Path（轨道路径），后3位 = Row（行编号） |
| 第4段 | 采集日期 | `20250415` | 卫星过境采集日期（UTC），YYYYMMDD 格式 |
| 第5段 | 处理日期 | `20250424` | USGS 处理生成日期，YYYYMMDD 格式 |
| 第6段 | 集合版本 | `02` | Collection 02（第二代数据集） |
| 第7段 | 质量等级 | `T1` / `T2` | T1 = Tier 1（最高质量），T2 = Tier 2 |
| 第8段 | 产品后缀 | `SR` | 仅 L2 产品带有 `_SR`（Surface Reflectance），L1 产品无此后缀 |

### 处理级别说明

| 级别代码 | 全称 | 含义 |
|---------|------|------|
| `L1TP` | Level-1 Precision Terrain (Corrected) | 精纠正 + 地形校正，使用地面控制点（GCP）和 DEM，几何精度最高 |
| `L1GT` | Level-1 Systematic Terrain (Corrected) | 系统几何校正 + 地形校正，使用 DEM 但无 GCP，精度略低于 L1TP |
| `L2SP` | Level-2 Surface Reflectance (Science Product) | 在 L1 基础上经大气校正生成地表反射率产品 |

### 质量等级说明

| 等级 | 含义 |
|------|------|
| `T1` (Tier 1) | 满足几何精度与辐射质量标准，可用于时间序列分析 |
| `T2` (Tier 2) | 质量略低于 T1（如云量过多、传感器异常），但仍可用于一般分析 |

## 文件命名规则

文件名格式为：`SceneID_波段标识.扩展名`

例如 L2 文件：
```
LC08_L2SP_119038_20250415_20250424_02_T1_SR_B1.TIF
└─────────────────── Scene ID ────────────────────┘  └─┘
                                                        波段标识 + 扩展名
```

例如 L1 文件：
```
LC08_L1TP_119038_20250415_20250424_02_T1_B8.TIF
└──────────────── Scene ID ────────────────┘  ─┘
```

## 波段文件详细说明

### Level-2 地表反射率产品（L2SP_SR 文件夹）

Level-2 数据经过大气校正，直接反映地表真实反射率，适合直接用于分类、植被指数计算和定量分析。每个 L2 场景包含以下文件：

| 文件名后缀 | 波段名称 | 波长范围 | 空间分辨率 | 说明 |
|-----------|----------|----------|------------|------|
| `_SR_B1.TIF` | Coastal Aerosol（海岸气溶胶） | 0.433–0.453 μm | 30 m | 蓝光波段，用于大气校正和浅水研究 |
| `_SR_B2.TIF` | Blue（蓝） | 0.450–0.515 μm | 30 m | 蓝光波段，对水体穿透力较强 |
| `_SR_B3.TIF` | Green（绿） | 0.525–0.600 μm | 30 m | 绿光波段 |
| `_SR_B4.TIF` | Red（红） | 0.630–0.680 μm | 30 m | 红光波段，对植被叶绿素敏感 |
| `_SR_B5.TIF` | Near Infrared（近红外） | 0.845–0.885 μm | 30 m | 近红外短波，常用于 NDVI 计算 |
| `_SR_B6.TIF` | SWIR 1（短波红外 1） | 1.560–1.660 μm | 30 m | 短波红外，用于土壤/植被水分识别 |
| `_SR_B7.TIF` | SWIR 2（短波红外 2） | 2.100–2.300 μm | 30 m | 短波红外，用于矿物识别 |
| `_SR_QA_AEROSOL.TIF` | Aerosol QA（气溶胶质量） | — | 30 m | 气溶胶类型与填充值质量标记 |
| `_QA_PIXEL.TIF` | Pixel QA（像元质量） | — | 30 m | 像元级质量波段，包含云、云阴影、雪/冰掩膜 |
| `_QA_RADSAT.TIF` | Radiometric Saturation QA（辐射饱和质量） | — | 30 m | 各波段辐射饱和状态检测 |

辅助文件：

| 文件名后缀 | 说明 |
|-----------|------|
| `_ANG.txt` | 太阳高度角、方位角、天顶角及传感器视角系数文件 |
| `_MTL.txt` | 场景元数据（文本格式），包含辐射校正、几何校正等关键参数 |
| `_MTL.xml` | 场景元数据（XML 格式），部分处理软件需要此格式 |
| `_MTL.json` | 场景元数据（JSON 格式），机器可读，便于程序解析 |

> **注意**：`_QA_PIXEL.TIF` 和 `_QA_RADSAT.TIF` 的文件名中**不包含** `SR_` 前缀，这是 USGS 的官方命名规范。

### Level-1 产品（L1TP / L1GT 文件夹）

Level-1 数据为原始数字值（DN），未经大气校正。本项目仅下载全色波段和热红外波段。每个 L1 场景包含以下文件：

| 文件名后缀 | 波段名称 | 波长范围 | 空间分辨率 | 说明 |
|-----------|----------|----------|------------|------|
| `_B8.TIF` | Panchromatic（全色） | 0.500–0.680 μm | 15 m | 全色波段，分辨率最高，常用于与多光谱融合 |
| `_B10.TIF` | TIRS 1（热红外 1） | 10.60–11.19 μm | 100 m | 热红外波段 1，用于地表温度反演 |
| `_B11.TIF` | TIRS 2（热红外 2） | 11.50–12.51 μm | 100 m | 热红外波段 2，可用于大气校正（劈窗算法） |

辅助文件：

| 文件名后缀 | 说明 |
|-----------|------|
| `_MTL.txt` | 场景元数据（文本格式） |
| `_MTL.xml` | 场景元数据（XML 格式） |
| `_MTL.json` | 场景元数据（JSON 格式） |

> **注意**：L1 产品**不包含** BQA 波段（实际下载配置中虽列出了 BQA，但部分场景可能未提供该文件）。L1 产品也**不包含** ANG.txt 文件。

## 空间分辨率对比

| 波段类型 | 分辨率 | 所属级别 | 说明 |
|---------|--------|---------|------|
| 全色 (B8) | 15 m | L1 | 分辨率最高，可用于图像融合提升多光谱分辨率 |
| 多光谱 (B1-B7) | 30 m | L2 | 地表反射率，最常用的分析波段 |
| 热红外 (B10, B11) | 100 m | L1 | 分辨率最低，用于地表温度反演 |
| QA 波段 | 30 m | L2 | 质量评估波段，用于掩膜去云 |

## 覆盖范围

- **省份**：浙江省全境
- **WRS-2 Path/Row**：13 组，覆盖浙江省范围的轨道条带：

  | Path | Row | 说明 |
  |------|-----|------|
  | 117 | 039, 040 | 浙江西部边缘 |
  | 118 | 038, 039, 040, 041 | 浙江中部主体 |
  | 119 | 038, 039, 040, 041 | 浙江中东部 |
  | 120 | 039, 040, 041 | 浙江东部沿海 |

- **卫星**：Landsat 8 (LC08) + Landsat 9 (LC09)
- **时间范围**：2025年4月 至 2025年9月（共 6 个月）
- **每月每 Path/Row 选 1 景**：按陆地云覆盖量（CLOUD_COVER_LAND）最低优先，云量相同时选最接近当月 15 日的场景
- **数据总量**：约 88 GB

## 数据来源

- **发现 API**：USGS Landsat STAC Server（https://landsatlook.usgs.gov/stac-server）
- **下载源**：AWS S3 公开存储桶 `s3://usgs-landsat/collection02/landsat/`
- **数据集合**：
  - Level-1：`landsat-c2l1`（Collection 2 Level-1）
  - Level-2 SR：`landsat-c2l2-sr`（Collection 2 Level-2 Surface Reflectance）

## 使用方法

```bash
# 完整下载
python main.py

# 仅预览（不实际下载）
python main.py --dry-run

# 指定时间范围
python main.py --start 202506 --end 202509

# 测试单个场景
python main.py --path 118 --row 039 --month 202504

# 重试失败的记录
python main.py --retry
```
